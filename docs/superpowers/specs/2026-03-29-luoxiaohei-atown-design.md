## 三、技术架构（方案 B：并发 tick + Memory Stream）

### 技术栈
- **后端**: Python 3.11 · FastAPI · asyncio · anthropic SDK
- **前端**: 单个 HTML 文件 · Canvas 2D · WebSocket
- **LLM**: claude-sonnet-4-6

### 目录结构
```
luoxiaohei-atown/
├── backend/
│   ├── main.py            # FastAPI app + WebSocket endpoint
│   ├── world.py           # WorldState, tick scheduler
│   ├── agents.py          # Agent class, Memory Stream, LLM prompt
│   ├── pathfinding.py     # A* on room graph
│   └── characters.py      # 4 角色初始设定（性格、位置、记忆）
├── frontend/
│   └── index.html         # Canvas渲染 + WebSocket客户端
└── docs/
    └── superpowers/specs/
        └── 2026-03-29-luoxiaohei-atown-design.md
```

### Tick 循环（每 4 秒 = 模拟 5 分钟）

```
Scheduler.tick()
  → 构建每个角色的 prompt（含 Memory Stream 最近 20 条）
  → asyncio.gather(agent_0.think(), agent_1.think(), ...)  # 4 并发
  → 每个 think() 调用 Claude → 返回 JSON
  → Pathfinder.compute(current_room, target_room)
  → 更新 WorldState
  → WebSocket 广播 tick 帧
```

### LLM Prompt 结构（每角色每 tick）

```
系统提示：你是[角色名]，性格[...]。现在是除夕夜[时间]，在哪吒家。
          用JSON回复，字段：思考/目标房间/动作/对话/情绪

上下文：
- 当前位置：[房间]
- 房间内有：[其他人名列表]
- 最近记忆：[Memory Stream 最近10条]
- 现在的情况：[WorldState 摘要]

请决定接下来5分钟你做什么。
```

### LLM 返回格式

```json
{
  "思考": "...",
  "目标房间": "客厅",
  "动作": "走向沙发坐下",
  "对话": "哪吒你在打什么游戏？",
  "情绪": "好奇"
}
```

### Memory Stream

每个 Agent 维护一个列表：
```python
memory: list[dict]  # 最多保留 20 条
# 每条: {"time": "18:30", "event": "无限做好了红烧肉", "importance": 8}
```

每次 tick 后，把本轮的动作和对话压入自己及同房间角色的记忆。

### A* 寻路图

房间视为节点，门视为边，带权重（距离）：
```
院子 - 厨房, 餐厅, 游戏室
厨房 - 院子, 餐厅, 衣帽间
餐厅 - 院子, 厨房, 游戏室, 客厅
游戏室 - 院子, 餐厅, 次阳台
衣帽间 - 厨房, 客厅, 卧室
客厅 - 餐厅, 衣帽间, 次阳台, 卧室
次阳台 - 游戏室, 客厅
卧室 - 衣帽间, 客厅, 卫生间
卫生间 - 卧室
```

前端根据路径帧序列做像素级插值动画（60fps，每房间移动约 0.8 秒）。

## 四、前端可视化

- **左侧主区**: Canvas 绘制平面图（基于 v5 SVG 设计稿）+ 角色精灵（简洁圆形色块+名字）
- **右侧面板**: 滚动日志，显示每角色的 思考/动作/对话
- **对话气泡**: 角色头顶弹出，3 秒后淡出
- **底部时间轴**: 显示当前模拟时间（18:00→23:00）
- **WebSocket 事件类型**:
  - `tick_start` — 新一轮开始
  - `agent_action` — 角色决策结果（含路径帧）
  - `dialogue` — 对话气泡触发
  - `world_update` — 世界状态更新

## 六、关键约束

- LLM 每 tick 最多 4 次并发调用，避免 rate limit 可加 0.5s jitter
- Memory Stream 超过 20 条时丢弃最旧的低 importance 事件
- 角色不能传送，必须经过寻路路径移动（即使目标就是当前房间也要"原地活动"）
- 对话只在同一房间或相邻房间的角色之间触发
